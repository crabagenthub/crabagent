# Token 优化策略（产品说明）

本文档描述 **Crabagent** 商业化能力之一：**在降低用户 Token / 成本的同时，尽量维持或提升会话效果**。  
实现细节与接口以 `architecture.md` 及后续 OpenAPI 为准；策略规则版本单独演进。

**关联文档**：[架构与数据流](./architecture.md)（含本能力与 Trace、Collector 的定稿边界）、[产品设计](./PRODUCT_DESIGN.md)、[技术设计](./TECHNICAL_DESIGN.md)、[文档索引](./README.md)。

---

## 1. 定位

| 能力 | 作用 |
|------|------|
| **Trace** | 观测与审计：钱花在哪、上下文如何增长、每跳模型与 Tool 行为 |
| **Token 优化** | 治理与优化：在观测基础上提供 **策略、建议、（分期）自动化**，目标为 **降本 + 不降质** |

**打包售卖叙事**：先看清（Trace），再自动省（Token 优化）。

---

## 2. 商业目标

1. **减少成本**：降低每轮 / 每会话的 input、cache 读写、无效 Tool 往返与 oversized 上下文。  
2. **提升会话效果**：在预算内保留对当前任务 **信息密度最高** 的上下文，减少噪声导致的跑偏、重复澄清。

---

## 3. 功能模块（分期）

### 3.1 诊断与基线（依赖 Trace / Usage）

- 按 **session、trace、模型、渠道** 展示 token 构成（system / history / tool / 图片等，以后端 schema 为准）。  
- **基线对比**：策略开启前后，同类型任务的 token / 估算费用 / 轮数（需聚合任务类型标签或客户自定义维度）。  
- **异常提示**：单轮暴涨、Tool 结果过大、重复 system、可摘要的长历史等（规则引擎 MVP；机器学习类后置）。

### 3.2 策略中心（核心产品形态）

策略为 **可开关、可组合、可覆盖范围** 的配置实体：

- **范围**：全局 / 按 agent / 按 channel / 按租户（与多租户模型对齐）。  
- **版本与回滚**：策略版本化、生效时间、一键回退。  
- **审计**：谁创建/修改、何时生效（与控制台合规能力一致）。

**策略类型示例**（首轮不必全部实现，按路线图取舍）：

| 类型 | 目的 | 说明 |
|------|------|------|
| 上下文预算 | 控上限 | turn / session 软上限；超限时裁剪顺序（如先摘要旧轮、再收缩 Tool 结果） |
| 历史压缩 / 摘要 | 省 long context | 与 compaction、context engine 理念对齐；模板 + 触发条件 |
| Tool 输出预算 | 省 input | 单工具最大字符、截断策略、引导模型分页读取 |
| 重复与缓存友好 | 省重复算 | 稳定 system 前缀、减少重复文档块；配合宿主 prompt cache 能力 |
| 模型与路由 | 性价比 | 简单任务小模型、复杂任务升配（需宿主 `before_model_resolve` 等配合） |
| 图片 / 附件 | 省 vision | 分辨率上限、仅首轮带图等 |

### 3.3 自动化执行（商业化溢价层）

| 阶段 | 行为 |
|------|------|
| **仅建议** | 控制台展示「预计可省比例/Token」+ 导出 **OpenClaw 配置片段**（`openclaw.json` 或文档说明） |
| **半自动** | 经授权的 **策略下发** 至网关侧配置（安全通道 + 客户显式同意） |
| **全自动（企业版）** | 与 **Context Engine / compaction** 深度集成；通常需 **官方或受信插件 + 宿主版本** |

### 3.4 效果与账单视角

- **仪表盘**：周 / 月 token、估算费用、策略命中次数、节省区间（估算需注明假设，如单价来自客户录入）。  
- **与 Trace 联动**：在 Trace 详情中高亮「本 trace 上可优化点」及当时生效策略（只读）。

---

## 4. 前端能力（在控制台增量）

1. **策略列表与编辑**：模板库、范围、优先级、冲突说明。  
2. **诊断页 / Trace 联动**：高亮过大 Tool、过长 history、建议摘要块等。  
3. **节省与效果**：预估 vs 实际（依赖后端聚合）。  
4. **（可选）推荐向导**：问答生成初始策略包，降低冷启动成本。  

**国际化**：页面标签与说明走 `next-intl`（中/英）；策略 **技术 key** 可保留英文，**展示名与描述**多语言。

---

## 5. 后端与数据

- **输入**：Trace 事件、usage 字段、（可选）OpenClaw 配置快照、客户填写的模型单价表。  
- **输出**：策略 CRUD、生效范围、只读 **建议 API**、（分期）**下发 API**。  
- **计算**：MVP 以 **规则引擎** 为主；批量报告可走异步任务 + 分析库（如 Parquet / DuckDB）。

---

## 6. 路线图建议

| 阶段 | 范围 |
|------|------|
| **V1** | 诊断（读 Trace/usage）+ **手工策略配置**（表单/JSON）+ **规则型节省估算** + 配置片段导出 |
| **V2** | 半自动下发、基线对比仪表盘强化 |
| **V3** | 与 context engine / compaction 深度联动、企业级自动优化 |

---

## 7. 风险与合规

- **「节省」数字**需在 UI 标注 **估算**及假设，避免财务纠纷。  
- **自动改写上下文或模型**可能影响行为，需 **显式 opt-in** 与 **回滚**。  
- **全量原文 Trace** 与 **策略建议** 同时存在时，注意 **最小权限**：策略引擎可仅用 **摘要 metrics** 而非全量 prompt。
