## 1. 产品概述

### 1.1 一句话定位

**面向 OpenClaw 及兼容变体的企业级可观测与成本治理套件**：在网关侧采集 **全链路 Trace**（含每跳模型请求与 Tool），在 **内网或云端控制台** 中检索、实时查看、按会话治理数据，并叠加 **Token 优化策略**（建议与分期自动化）以降低模型成本、改善长会话效果。

### 1.2 产品愿景

- **可观测**：团队能还原单次助手运行的输入、上下文、模型调用、工具调用与压缩过程，支撑排障、合规与效果分析。  
- **可治理**：在观测基础上提供 **策略化** 的上下文与成本建议，与「已有商业化方案」对齐的 **降本 + 提质** 叙事。  
- **可落地**：以 **独立 npm 插件 + Collector + Web** 交付，适配 **全局 npm 安装的 OpenClaw**，变体宿主可 **能力降级** 仍可用。

### 1.3 非目标（当前阶段明确不做）

- 替代 OpenClaw 本体或修改其安全敏感核心代码。  
- 在浏览器内直接管理用户本机 OpenClaw 配置文件路径（属运维与网关侧职责）。  
- MVP 内对 OpenClaw 运行时 **自动下发并改写** 模型/上下文（Token 优化 V1 仅为建议 + 配置导出，见架构定稿 §4.6）。

## 2. 目标用户与使用场景

### 2.1 用户角色

| 角色 | 诉求 |
|------|------|
| **平台 / 运维** | 部署插件与 Collector，配置网络与密钥，监控采集健康与队列积压。 |
| **开发 / Agent 负责人** | 排查单次失败或效果差、对比 compaction 前后、看 Tool 与多跳模型。 |
| **管理者 / 采购** | 看 Token 与估算成本、评估 Token 优化策略带来的节省（估算需标注假设）。 |
| **合规 / 安全** | 按会话删除数据、审计谁访问策略与配置、数据驻留内网。 |

### 2.2 典型场景

1. **排障**：用户反馈「答非所问」，在 Trace 时间线中查看当轮 `llm_input`、每跳 `model_call`、Tool 入出参摘要。  
2. **成本归因**：按会话 / 模型 / 渠道聚合 usage，识别 oversized Tool 或过长 history。  
3. **压缩审计**：展示 compaction 前后差异，解释「为什么上下文变短了」。  
4. **子任务**：父会话中启动子代理，**同一主时间线** 查看父子关系（`trace_root_id` 合并展示）。  
5. **降本试点**：启用 Token 优化 **建议**，导出 OpenClaw 配置片段，对比试点前后基线（V2 强化仪表盘）。

## 3. 价值主张与差异化

| 支柱 | 用户收益 |
|------|-----------|
| **Trace** | 比仅看会话文本更细：每跳模型、Tool、compaction、（可选）wire 级上下文摘要。 |
| **变体友好** | 官方 OpenClaw + fork：无 per-model 能力时 **降级展示**，不阻断采集。 |
| **离线韧性** | 网关侧 **SQLite Outbox**，断网续传，适合内网不稳定环境。 |
| **Token 优化** | 与 Trace **同一数据面**：诊断 → 策略 →（分期）自动化，商业上可组合售卖。 |
| **部署灵活** | **公有云 SaaS** 与 **私有化** 同一套控制台能力；私有化支持 **GitHub 等 OAuth** 与 **API Key**。 |

## 4. 功能需求

### 4.1 Trace 与观测（MVP）

| ID | 功能 | 说明 | 优先级 |
|----|------|------|--------|
| T-01 | Trace 列表 | 分页、排序、筛选（时间、session、错误态等，以后端能力为准） | P0 |
| T-02 | Trace 详情 / 时间线 | 主时间线：turn、模型调用（多段，依赖 `per_model_call`）、Tool、compaction 节点 | P0 |
| T-03 | 能力降级提示 | 展示 `capabilities.per_model_call` 等；无则提示「仅 turn 级模型块」 | P0 |
| T-04 | Session 视图 | 按 `sessionId` 查看关联 Trace / 事件 | P0 |
| T-05 | 按 session 删除 | 级联删除该会话相关数据，二次确认 | P0 |
| T-06 | 实时详情 | **SSE** 订阅 `trace_root_id` 流；列表以轮询为主 | P0 |
| T-07 | Compaction 展示 | before/after 条数与摘要 / diff（与后端字段对齐） | P0 |
| T-08 | 子代理合并 | 同一 `trace_root_id` 下父子 span 一条时间线 | P0 |

### 4.2 认证与租户（MVP）

| ID | 功能 | 说明 | 优先级 |
|----|------|------|--------|
| A-01 | 部署模式 | `cloud` / `self-hosted` 控制菜单与部分流程 | P0 |
| A-02 | 公有云人机登录 | 租户账号（具体形态：邮箱密码 / Magic link 等实现阶段定） | P0 |
| A-03 | 私有化 OAuth | 如 GitHub：**Refresh + HttpOnly Cookie**，减少重复登录 | P0 |
| A-04 | 私有化降级 | 无公网 IdP 时：客户 IdP / 仅 API Key / 可选本地管理员 | P1 |
| A-05 | API Key | 创建、展示（仅一次）、吊销；供插件与脚本 | P0 |
| A-06 | 登出 | 清除会话，跳转登录 | P0 |

### 4.3 Web 基础体验（MVP）

| ID | 功能 | 说明 | 优先级 |
|----|------|------|--------|
| W-01 | 国际化 | **中文、英文**；Trace 内用户原文不翻译 | P0 |
| W-02 | 语言切换 | Header 或设置，持久化 | P0 |
| W-03 | 空状态与错误 | 无数据、401/403、Collector 不可达 | P0 |
| W-04 | 响应式 | 桌面优先；表格与时间线窄屏可滚动 | P1 |

### 4.4 Token 优化（与 [product-token-optimization.md](./product-token-optimization.md) 一致）

| 阶段 | 功能摘要 | 优先级 |
|------|-----------|--------|
| **V1** | 策略 CRUD、范围与版本；规则型诊断与 **节省估算**；**OpenClaw 配置片段导出**；与 Trace 详情联动高亮可优化点 | P1（可与 Trace MVP 并行或紧随其后） |
| **V2** | 基线对比仪表盘、半自动策略下发 | 路线图 |
| **V3** | 与 context engine / compaction 深度联动 | 路线图 |

### 4.5 插件侧（产品对客户的承诺边界）

- 采集 **Hook** 所列生命周期事件；**每跳模型**依赖宿主 **Cache Trace**（或变体等价能力），否则产品侧 **降级说明** 而非静默丢失解释。  
- **按 session 采样**：未命中会话 **几乎零开销**（Hook 内快速 return）。  
- **原文全量上送**在合规前提下提供；客户需知磁盘与带宽占用。

## 5. 非功能需求

### 5.1 性能与可靠性

- **网关**：Hook 路径 **禁止阻塞网络**；内存队列有界，背压策略可配置（丢弃最旧或转 Outbox 由技术设计细化）。  
- **Collector**：ingest 支持批量与压缩；SQLite WAL；SSE 连接在代理上关闭缓冲。  
- **可用性目标**：内网部署以 **可恢复** 为先（Outbox 续传），不承诺与公网 SaaS 同级 SLA（可在合同中单独约定）。

### 5.2 安全与合规

- 传输 **TLS**；控制台与 ingest **鉴权分离**（Cookie 会话 vs API Key）。  
- **按 sessionId 删除** 满足客户「右被遗忘」类诉求的落地入口（法律效力以合同为准）。  
- Token 优化 **「节省」数字** 必须 UI 标注 **估算**及假设。

### 5.3 国际化与可访问性

- UI 字符串全走 i18n；日期数字用 `Intl`。  
- MVP 可访问性：键盘可操作主要流程、对比度符合内网规范（如有）。

## 6. 与 OpenClaw 的关系

- **宿主**：用户本机或服务器上 **全局 npm** 安装的 `openclaw`（或变体）。  
- **集成方式**：`plugins.load.paths` 指向 Crabagent 插件包；可选开启 **Cache Trace** 以满足 per-model 采集。  
- **边界**：Crabagent **不**替代 OpenClaw 路由、渠道与模型注册表；仅 **旁路观测与策略产品**。

## 8. 术语表

| 术语 | 含义 |
|------|------|
| **Trace** | 一次用户任务在主时间线上的可观测记录，以 `trace_root_id` 为根。 |
| **sessionId** | OpenClaw / Pi 侧会话实例 ID。 |
| **trace_root_id** | Crabagent 侧主 Trace 根 ID；子代理事件挂其下。 |
| **Collector** | 接收 ingest、存 SQLite、提供 REST 与 SSE 的后端服务。 |
| **ingest** | 插件向 Collector 批量上报事件的接口。 |
| **Outbox** | 网关本机 SQLite 中待重试的上报队列。 |
| **能力降级** | 宿主无 Cache Trace 等能力时，控制台明确提示并减少 per-model 展示。 |
